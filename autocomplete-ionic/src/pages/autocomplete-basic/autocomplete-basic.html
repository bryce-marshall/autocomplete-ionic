Autocomplete Basic City In this example the bound model data-item is a simple string (the city name) and therefore the query function returns an array of string instances. The allowCreate property is set to "false" and the autoAssign property is set to "null" (a string, and not the typescript/ECMAs script null operator). This means that the user:
  • MUST assign the field with an item from the popup list;
  • MUST explicitly click on an item in the list to select it (it will not auto-assign when the control loses focus);
  • but MAY auto-assign a null/empty value to the field, which will be applied when the control loses focus.
